## 1. 总体概述
### 目标
输入英文单词，自动匹配可用趣味记忆方案；支持多方案组合输出、按人群动态调整权重，辅助用户快速记忆单词拼写与词义。
### 适用场景
单词详情页（单个单词）、批量单词学习模式。
### 人群分层
- 低龄小学生：偏重谐音、场景联想、顺口溜、复合词
- 初高中/成人：偏重词根词缀、复合词、词中词、形近对比

## 2. 方案优先级调度策略
程序匹配顺序（从高到低）
1. 词根词缀记忆法
2. 复合词拆分记忆法
3. 词中词寻宝记忆法
4. 形近词对比记忆法
5. 谐音联想记忆法
6. 图像场景联想记忆法
7. 成对关联记忆（近义/反义）
8. 主题归类打包记忆
9. 字母特征记忆法
10. 短句微型故事串联记忆
11. 押韵顺口溜记忆

> 输出策略：单个单词最多返回**2~3种不同类型方案**，避免同类型方案重复。

## 3. 十一大记忆规则完整定义
### 规则1｜词根词缀记忆法（长效记忆 · 一级方案）
**定义**
利用英语构词单元：前缀、词根、后缀拆解单词，通过构词逻辑理解词义，适合中长单词。
**判定条件**
单词可切分为前缀+词根+后缀；词库具备标准词根词缀释义；禁止虚构词缀。
**生成逻辑**
1. 拆分单词结构
2. 分别解释前缀/词根/后缀含义
3. 组合推导单词本义
4. 可选增加通俗联想话术
**示例**
`unhappy = un(不)+happy(开心) → 不开心`
`transport = trans(跨越)+port(搬运) → 跨地域搬运→运输`
**约束边界**
1. ≤4个短字母单词不优先启用
2. 不规则外来词汇不强行套用
3. 严禁编造不存在词根、词缀

### 规则2｜复合词拆分记忆法（高通用 · 一级方案）
**定义**
单词由两个独立完整英文单词拼接构成，拆分后借助熟词理解整体词义。
**判定条件**
单词可切分为两段，两段均为合法独立英文词汇。
**生成逻辑**
单词 = A单词 + B单词；分别释义，构建联想串联词义。
**示例**
`pineapple = pine(松树)+apple(苹果) → 菠萝`
`bedroom = bed(床)+room(房间) → 卧室`
**约束边界**
1. 拆分片段必须为标准词汇，禁止无意义切割
2. 区分复合词与普通单词，不可强行切割（mother 不能拆 mot+her）

### 规则3｜词中词寻宝记忆法（内嵌熟词联想）
**定义**
单词字符串内部包含高频熟词，利用已知熟词搭建联想桥梁。
> 区分：复合词是词源本身拼接；词中词仅字符串包含，词源不一定相关。
**判定条件**
目标单词存在连续子串为常用基础单词。
**生成逻辑**
提取内嵌熟词，编写简短故事关联单词整体含义。
**示例**
`friend 内含 end（终点）`
联想：真正的朋友不会走到终点。
`season 内含 sea（大海）`
**约束边界**
1. 优先匹配≥3字母内嵌词
2. 规避 a/an/is 等超短虚词

### 规则4｜形近词对比记忆法（防混淆专用）
**定义**
筛选拼写高度近似、仅相差1~2个字母单词，成对捆绑，标记差异点区分记忆。
**判定条件**
1. 单词长度相同或相差1
2. 编辑距离 ≤ 2
3. 词汇属于高频词库
**细分类型**
1. 增字母：plan → plane(+e)
2. 减字母：through → though
3. 字母调换：quiet / quite
4. 单字母替换：cat / bat / mat
**生成逻辑**
高亮差异字母，搭配区分联想口诀。
**约束边界**
单次对比单词数量不超过3个，避免信息过载。

### 规则5｜谐音联想记忆法（强趣味 · 短时记忆）
**定义**
利用单词读音与中文发音近似，构建画面联想辅助记忆拼写。
**判定条件**
音标读音匹配通顺中文谐音；读音差距过大直接放弃。
**生成逻辑**
提取谐音，搭配简短故事绑定单词释义。
**示例**
`ambulance 俺不能死 → 救护车`
`pest 拍死它 → 害虫`
**强制约束**
1. 必须附带提示：谐音仅辅助记忆拼写，不可模仿发音
2. 禁用低俗、负面、违和谐音
3. 不可单独作为唯一记忆方案，尽量搭配其他方法

### 规则6｜图像场景联想记忆法（右脑画面记忆）
**定义**
不拆分拼写，围绕词义构建可视化场景画面。
**判定条件**
名词、动作动词、具象形容词优先；抽象词汇不单独使用。
**生成逻辑**
构造画面：人物+动作+环境。
**示例**
`elephant`：长鼻子、大耳朵的巨型大象
`umbrella`：下雨天撑开雨伞遮挡雨水
**约束边界**
抽象词汇（freedom、honest）建议搭配其他记忆方案。

### 规则7｜成对关联记忆（反义词 / 近义词）
#### 7.1 反义捆绑记忆
判定：词库存在标准反义词
逻辑：成对记忆，建立对立关联
示例：hot ↔ cold；buy ↔ sell

#### 7.2 近义捆绑记忆
判定：词义高度相近
示例：happy / glad；big / large
**约束**
优先高频词汇，不一次性罗列大量近义词造成混淆。

### 规则8｜主题归类打包记忆（批量学习专用）
**定义**
同一主题单词组成词汇集合，适合批量背诵场景。
**判定条件**
批量记词模式开启；单词详情页可作为「同类词汇推荐」拓展。
**常见分类维度**
动物、食物、方位、情绪、交通工具、职业、颜色等。
**输出形式**
【同类词汇】tiger、lion、rabbit、monkey

### 规则9｜字母特征记忆法（补充备用方案）
**定义**
利用单词字母特征：重复字母、首尾特征、特殊字母组合。
**示例**
`banana` 多次重复字母a
`see` 两个e，代表两只眼睛→看见
**约束边界**
仅作为补充方案，不单独主推展示。

### 规则10｜短句/微型故事串联记忆
两种模式：
1. 单词基础短句造句
`cat：The cat likes fish.`
2. 多单词串联小故事（仅批量模式启用）
**判定条件**
批量单词学习场景；单个单词页面仅作拓展内容。

### 规则11｜押韵顺口溜记忆（低龄小学生专属）
**定义**
利用英文尾音押韵编写简易短句，朗朗上口。
**示例**
bee & tree：I see a bee fly to the tree.
**约束边界**
成人模式降低权重，优先展示逻辑类记忆方案。

## 4. 系统调度约束规则（后端核心逻辑）
1. 同一个单词不输出同类型多条记忆方案，自动去重
2. 谐音方案若谐音牵强、读音偏差大，直接舍弃
3. 严禁虚构词根、强行无逻辑切割单词
4. 联想文案长度控制：单条文案 20～80 字
5. 黑名单拦截：暴力拆分（如 teacher 强行拆 tea+cher）直接过滤
6. 用户分层动态权重
    - 小学生模式：提升谐音、顺口溜、场景联想、复合词权重；降低复杂词根词缀权重
    - 初高中/成人模式：提升词根词缀、形近对比、词中词权重

## 6. 伪代码参考逻辑
```javascript
function getWordMemoryMethod(word) {
    let methodList = []
    if (canRootAffixSplit(word)) {
        methodList.push(createRootAffixMethod(word))
    }
    if (isCompoundWord(word)) {
        methodList.push(createCompoundMethod(word))
    }
    if (hasInnerKnownWord(word)) {
        methodList.push(createInnerWordMethod(word))
    }
    if (hasSimilarSpellingWord(word)) {
        methodList.push(createSimilarCompareMethod(word))
    }
    if (goodHomophonic(word)) {
        methodList.push(createHomophonicMethod(word))
    }
    if (isConcreteWord(word)) {
        methodList.push(createSceneImageMethod(word))
    }
    if (hasAntonym(word)) {
        methodList.push(createPairMethod(word))
    }
    // 去重，截取最多3条方案返回
    return unique(methodList).slice(0, 3)
}
```
